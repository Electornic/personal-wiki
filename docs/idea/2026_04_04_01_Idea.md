# 2026_04_04_01_Idea

## Note

- 이 문서는 버전 범위 확정 문서가 아니라 아이디어 백로그 메모다.
- 외부 디자인 리뷰 피드백을 기반으로 정리한 UI/UX 개선 항목이다.

## Status

- `backlog`
- 0.7.x 패치에서 순차적으로 반영 예정.

## Problem

- 앱의 의도를 모르는 외부 사용자가 UI를 직관적으로 이해하지 못하는 부분이 다수 존재한다.
- 위험한 액션(취소, 삭제)에 대한 보호 장치가 없다.
- 상태 구분(체크 여부, public/private, book/article)이 시각적으로 불분명하다.

## Idea

### 홈
- Hero 영역("A Space for Reading & Reflection")이 너무 크다. 첫 방문 시에만 보이거나 더 간략하게 줄인다.

### 에디터
- 헤더 버튼이 너무 작다. 터치/클릭 영역 확대 필요.
- Cancel 버튼이 너무 쉽게 눌린다. 작성 중 내용이 있으면 확인 팝업을 띄운다.
- Publish Record 버튼을 상단에서 하단(글 작성 완료 후)으로 이동한다.
- Writing support 섹션의 의도가 앱을 모르는 사용자에게 읽히지 않는다. 설명을 개선하거나 위치를 재고한다.

### Workspace (문서 목록)
- Private 배지 — 빨간 배지 대신 자물쇠 아이콘만으로 충분할 수 있다. Tag와 색상이 겹쳐 혼란.
- 카드 전체가 클릭 가능해야 한다. 별도 "자세히 보기" 버튼 불필요.
- 삭제가 바로 실행된다. 확인 팝업을 추가한다.

### Bookmark / Like
- 체크 여부(활성/비활성) 구분이 안 된다. filled 아이콘(활성) / outline 아이콘(비활성)으로 명확히 구분한다.

### 문서 카드
- 작성자가 "unknown"으로 표시되면 혼란 — "뒤비갓이 책을 쓴 사람인가요?" 같은 오해 발생. 작성자 표시 로직 정리 필요.
- book 아이콘과 article 아이콘의 차이가 직관적이지 않다. 아이콘 또는 레이블로 구분을 명확히 한다.
- 글이 없는 문서 카드가 허전하다. 빈 상태 처리 개선.

## Why It Might Matter

- 외부 사용자의 첫인상에서 제품의 완성도가 결정된다.
- 위험 액션 보호는 기본 UX 품질 요소다.
- 상태 구분이 명확해야 개인 도서관으로서 신뢰감이 생긴다.

## Risks / Open Questions

- 항목이 많아 한 번에 다 하면 범위가 커진다. 우선순위를 정해 0.7.x 패치로 나눠야 한다.
- Hero 영역 축소가 홈 페이지의 성격을 바꿀 수 있다. 어느 정도까지 줄일지 결정 필요.
- Writing support 섹션을 없앨지, 개선할지, 챗 기능으로 대체할지 판단 필요.

## Promotion Rule

- 항목별로 묶어 0.7.x 패치 task로 승격한다.
- 챗 기능(0.7.0) 완료 후 순차 진행.
