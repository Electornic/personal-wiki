# 2026_03_14_02_Idea

## Note

- 이 문서는 버전 범위 확정 문서가 아니라 아이디어 백로그 메모다.

## Problem

- 데스크탑 기준으로 header width와 body width가 통일되지 않아 제품이 분리되어 보인다.
- library 페이지와 write/edit 페이지의 컨테이너 폭이 달라 화면 전환 시 어색함이 생긴다.
- 페이지 이동 시 `0.7~0.8s` 수준으로 느껴져 실제 속도와 별개로 체감이 둔하다.
- related reading 영역이 다음 문서로 이동할 이유를 충분히 설명하지 못한다.

## Idea

- 데스크탑 기준 header/body width alignment를 먼저 정리한다.
- loading UI, prefetch, 캐시 전략을 포함해 체감 속도를 개선한다.
- 관련 문서에 왜 추천되었는지 공통 태그나 주제를 함께 보여준다.
- 태그 필터, 정렬, 간단 검색을 추가해 문서 간 이동을 더 쉽게 만든다.
- 읽기 화면과 같은 여백, 타이포, 폭 규칙을 write/edit에도 맞춰 제품 톤을 통일한다.
- 관련 문서가 없을 때는 empty state와 다음 행동 유도 문구를 더 분명하게 둔다.
- 작성자가 현재 문서의 visibility를 바로 이해할 수 있게 배지와 상태 표시를 강화한다.

## Why It Might Matter

- 읽기와 작성 화면의 밀도를 맞추면 하나의 제품처럼 느껴진다.
- 체감 속도 개선은 기능 추가 없이도 제품 품질을 바로 끌어올린다.
- related reading의 연결 근거를 분명히 하면 개인 도서관형 위키의 탐색성이 살아난다.

## Risks / Open Questions

- 디자인 정리와 속도 개선 중 어느 쪽을 먼저 잠글지 결정이 필요하다.
- 태그 필터와 간단 검색이 실제 버전 범위로 승격될지 아직 정해지지 않았다.
- related reading 개선이 recommendation 알고리즘 변경까지 요구하는지는 추가 판단이 필요하다.
