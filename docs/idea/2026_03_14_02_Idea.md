# 2026_03_14_02_Idea

## Note

- 이 문서는 버전 범위 확정 문서가 아니라 아이디어 백로그 메모다.

## Current Frictions

- 디자인 개선
  - 데스크탑 기준으로 header width와 body width를 통일한다.
  - library 페이지와 write/edit 페이지의 컨테이너 폭이 달라 어색한 문제를 먼저 정리한다.
- 속도 개선
  - 페이지 이동 시 0.7~0.8초 수준으로 느껴지는 전환 속도를 줄인다.
  - 실제 응답 속도와 별개로 loading UI, prefetch, 캐시 전략 등 체감 속도도 함께 개선한다.

## Additional Ideas

- 읽기 흐름 개선
  - 관련 문서에 왜 추천되었는지 공통 태그나 주제를 함께 보여준다.
- 라이브러리 탐색성 개선
  - 태그 필터, 정렬, 간단 검색을 추가해 문서 간 이동을 더 쉽게 만든다.
- write/edit 경험 정리
  - 읽기 화면과 같은 여백, 타이포, 폭 규칙을 써서 제품 톤을 통일한다.
- 추천 empty state 개선
  - 관련 문서가 없을 때 비워두지 말고 명확한 안내 문구를 둔다.
- 공개/비공개 상태 인지성 개선
  - 작성자가 현재 문서의 visibility를 바로 이해할 수 있게 배지와 상태 표시를 강화한다.
