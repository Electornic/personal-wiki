## Problem

- 현재 제품은 reading, writing, reactions, reading flow의 기반은 갖췄지만, 문서 간 탐색과 재방문 흐름은 더 확장할 여지가 있다.
- 아이디어를 기능별로 쪼개기 전에 한 곳에서 backlog로 길게 모아보고 우선순위를 다시 잡을 필요가 있다.

## Idea

- 태그 필터 추가
  - library나 my library에서 태그 기준으로 문서를 빠르게 좁혀볼 수 있게 한다.
- 정렬 옵션 추가
  - 최신순, 오래된 순, 최근 수정순, 반응 많은 순 같은 기본 정렬을 제공한다.
- 간단 검색
  - 제목, writer, tag 기준의 가벼운 검색을 추가한다.
- 주제 허브 페이지
  - 특정 태그나 주제를 눌렀을 때 관련 문서와 대표 메모를 묶어서 보여주는 탐색 페이지를 만든다.
- 추천 이유 노출 강화
  - related reading에 왜 추천되었는지 같은 태그, 같은 주제 같은 근거를 더 분명하게 보여준다.
- 추천 카드 메모 미리보기
  - related reading 카드에 대표 thought나 메모 한 줄을 같이 보여준다.
- related reading empty state 고도화
  - 비어 있을 때 단순 안내가 아니라 다른 탐색 진입점을 같이 보여준다.
- 다음 읽기 1개 제안
  - 여러 개를 나열하는 대신 다음 문서 하나를 강하게 추천하는 흐름을 만든다.
- 문서 간 관계 타입
  - 두 문서를 `확장`, `반박`, `보충`, `입문` 같은 관계로 연결할 수 있게 한다.
- 메모-메모 연결
  - 문서 단위가 아니라 특정 note/thought 단위끼리 연결한다.
- 큐레이션 선반
  - `입문용`, `같이 읽으면 좋은 3개`, `이번 달 다시 읽은 글` 같은 수동 큐레이션 묶음을 만든다.
- 재방문 흐름
  - 예전에 읽은 문서를 다시 꺼내보는 revisit queue나 resurfacing을 만든다.
- 읽은 맥락 기록
  - 왜 읽었는지, 어떤 문제의식으로 읽었는지, 나중에 다시 볼 이유가 뭔지 같은 메타를 기록한다.
- 문서 상태 인지성 강화
  - 작성자가 현재 문서의 public/private 상태를 더 분명히 인식하도록 배지와 상태 표시를 강화한다.
- reading progress 힌트
  - 긴 글에서 현재 위치나 주요 섹션 흐름을 가볍게 보여준다.
- 대표 문장 / 대표 메모 pinning
  - 문서마다 핵심 thought 하나를 대표로 고정해서 목록이나 추천에서 먼저 노출한다.
- library landing 큐레이션
  - 홈에서 단순 최신 목록 외에 주제별 섹션이나 curated row를 같이 보여준다.
- my library 정리 기능
  - bookmark/like 외에 revisit, archive, remove 같은 개인 관리 축을 더한다.
- 반응 기반 보조 정렬
  - 추천 기준 자체를 바꾸지 않되 tie-breaker로 like/bookmark를 약하게 반영한다.
- source context 보강
  - book/article의 최소 메타를 더 잘 보이게 하되 현재 단순 record 모델을 해치지 않는 방향으로 정리한다.
- 문서 묶음 reading path
  - A -> B -> C 순서로 이어 읽는 경로를 author가 직접 저장한다.
- 관련 문서 그룹화
  - related reading을 `같은 주제`, `다른 관점`, `다음 단계`처럼 나눠 보여준다.
- writer thought timeline
  - 시간 순으로 다시 본 생각, 수정된 생각을 보는 흐름을 만든다.
- discovery cue 강화
  - tag, writer, relation, highlighted memo를 더 적극적인 탐색 단서로 활용한다.
- 홈 empty state / sparse state 개선
  - 문서 수가 적을 때도 허전하지 않게 제품 성격이 보이는 화면을 만든다.
- onboarding copy 정리
  - sign-in, workspace, editor, public reading에서 제품의 쓰임새를 더 자연스럽게 설명한다.
- lightweight analytics for curation
  - 어떤 문서에서 다음 문서로 많이 넘어가는지 정도만 내부적으로 보는 최소 분석을 둔다.
- 추천 실패 케이스 기록
  - related reading이 자주 비는 태그나 문서 유형을 backlog로 다시 남겨 후속 큐레이션 기준으로 쓴다.

## Why It Might Matter

- 위키의 핵심 가치인 `한 문서를 읽다가 다음 문서로 자연스럽게 이동하는 경험`을 더 강하게 만들 수 있다.
- 자동 추천만으로 부족한 부분을 주제 허브, 큐레이션, 관계 링크 같은 구조로 보완할 수 있다.
- bookmark, like, my library 이후의 개인 사용 흐름을 더 길게 이어갈 수 있다.
- 아이디어를 한 문서에 모아두면 바로 task로 승격할 항목과 오래 backlog에 둘 항목을 구분하기 쉬워진다.

## Risks / Open Questions

- 추천, 검색, 큐레이션, 관계 링크를 한 번에 확장하면 범위가 너무 커질 수 있다.
- 탐색성 개선이 recommendation 알고리즘 변경까지 요구하는지는 항목마다 따로 판단해야 한다.
- 어떤 아이디어는 author-only 도구에 가깝고, 어떤 아이디어는 public reading 경험에 더 직접적이라 우선순위 충돌이 생길 수 있다.
- analytics나 revisit 같은 항목은 제품 철학에 맞게 최소 수준으로 제한할지 먼저 정해야 한다.

## Promotion Rule

- 한 아이디어가 별도 버전 범위로 묶일 만큼 우선순위가 높아지고, 구현 범위와 acceptance criteria가 분명해지면 `docs/task` 문서로 승격한다.
- 이미 다른 task로 일부 반영된 항목은 중복 task를 만들지 않고, 해당 task의 후속 backlog로 연결한다.
