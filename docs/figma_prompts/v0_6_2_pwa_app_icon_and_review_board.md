# v0.6.2 PWA App Icon And Review Board

## Prompt

Design the `Personal Wiki` v0.6.2 PWA app icon set using Figma Make.

This is not a request for a full product page redesign.
It is a focused icon-design task for the installable web app experience.

This prompt covers:
1. PWA app icon exploration
2. icon comparison board
3. small-size readability check
4. iOS / Android icon crop review

## Product Context

The product is a personal library-style wiki.

Core product direction:
- reading-first
- calm and editorial
- quiet, reflective, library-like
- not dashboard-heavy
- not productivity-gadget styled

The key experience is:
- reading one record
- then moving naturally into related records

Important mood:
- warm
- literary
- tactile but minimal
- focused on books, notes, margins, bookmarks, and connected reading

Avoid making the icon feel like:
- a SaaS dashboard
- a generic notes app
- a to-do app
- a gaming icon
- a loud social app

## Main Goal

Create a set of PWA app icon directions that feel appropriate for:
- a personal reading tool
- a quiet knowledge library
- an installable reading experience on mobile and desktop

The icon should feel:
- clear
- memorable
- calm
- strong at small sizes

The icon should not rely on:
- text labels
- tiny details
- decorative complexity that disappears when scaled down

## What To Produce

### 1. App icon concepts

Create `3` icon directions.

Each direction should be designed as if the master source is:
- `1024 x 1024`

Each icon should be:
- symbol-first
- readable at very small size
- strong enough for a home screen icon

Suggested direction names:
- `Quiet Book`
- `Bookmark Frame`
- `Reading Window`

You may rename them if you find better names.

### 2. Comparison / review board

Create one review board page that shows:
- all 3 icon directions side by side
- small-size previews
- iOS rounded-crop preview
- Android maskable-crop preview
- a light home-screen style mock for context

This review board is important.
It should help decide:
- which icon survives best at small sizes
- which icon fits the product tone best
- which icon feels most installable and app-like

## Design Direction

Prefer symbols related to:
- book
- page
- bookmark
- margin
- reading card
- quiet sequence / continuation

Possible visual ideas:
- an open book reduced to a simple geometric shape
- a closed book with one visible bookmark tab
- a page window / reading frame
- layered card/page forms implying connected reading

Avoid:
- checkmarks
- clipboards
- chat bubbles
- folder-only metaphors
- heavy gradients
- over-illustrated objects

## Color Direction

Use a palette that fits the current product:
- warm paper-like tones
- dark ink brown / near-black for contrast
- muted editorial neutrals

Good references for feel:
- paper
- ink
- bookmark cloth
- warm shelf wood

Important:
- the icon must still have strong contrast on a home screen
- avoid overly pale solutions that disappear into light backgrounds

## Shape And Readability Requirements

Please optimize for:
- `48 x 48`
- `64 x 64`
- `96 x 96`
- `192 x 192`
- `512 x 512`

The icon should still read clearly at small size.

Please explicitly check:
- silhouette clarity
- center-weight balance
- safe area for rounded crop
- safe area for maskable crop

## Platform Review Needs

### iOS

Show how each icon feels under a rounded-square crop.

### Android

Show how each icon feels in a maskable crop.

The icon should not lose its main symbol when cropped.

## Output Expectation

Produce:
- `3` app icon concepts
- `1` comparison/review board

The review board should include:
- direction name
- master icon preview
- small-size previews
- iOS crop preview
- Android maskable preview
- one short note about what each direction communicates

## Decision Criteria

The best direction will be the one that most strongly feels like:
- a reading tool
- a quiet personal library
- an editorial knowledge product

More important than “beautiful illustration”:
- recognizability
- small-size clarity
- product fit
- installable app presence

## Final Use Context

The chosen icon will later be exported into real PWA assets such as:
- `192 x 192`
- `512 x 512`
- `512 x 512 maskable`
- `180 x 180 apple touch icon`

So please design with real export usage in mind, not only presentation.
