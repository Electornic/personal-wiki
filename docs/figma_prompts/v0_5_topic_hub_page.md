# v0.5 Topic Hub Page

## Prompt

Design the `Personal Wiki` v0.5 topic hub page.

This is not a full redesign of the product.
It is a new exploration surface inside the existing warm, editorial `Personal Wiki` visual system.

This prompt is specifically for the `topic hub page`.
Do not redesign:
- the full home page
- the full record detail page
- the workspace
- the editor

## Product Context

The product already supports:
- public reading
- authenticated writing/editing
- comments and replies
- bookmark / like reactions
- `My Library`
- related reading on record detail
- discovery controls such as filter / sort / search in library surfaces

Current product direction:
- reading-first
- literary and editorial, not dashboard-like
- recommendations are still based on shared tags/topics
- private records must never leak through public surfaces

The `topic hub` should help readers move from:
- one tag
- one theme
- one cluster of records

into a deeper reading path.

## Main Goal

Create a dedicated page for exploring one topic/tag in a way that feels:
- curated
- calm
- easy to scan
- better than a plain filtered list

The page should make one topic feel like:
- a reading shelf
- a thematic table
- a focused path through the library

not:
- a search results page
- a recommendation feed
- a noisy taxonomy screen

## What This Page Needs To Do

The topic hub page should let a reader:
- understand which topic they are currently exploring
- see why this topic matters in the context of the library
- browse all records connected to that topic
- quickly identify a good entry point
- continue into related records naturally

Optional but useful:
- highlight one representative memo, quote, or idea fragment
- surface one curated shelf related to the topic

## Core Content Blocks

### 1. Topic header

Include:
- topic/tag title
- short introductory sentence or context line
- count of connected records if it helps

Tone:
- calm
- editorial
- not taxonomy-heavy

### 2. Featured topic entry

This section should answer:
- “Where should I start if I want to understand this topic?”

Possible content:
- a featured record
- representative memo preview
- short explanation of why this record matters for the topic

This should feel like a soft recommendation, not a loud CTA block.

### 3. Topic record list

Show records connected to the topic.

Each item should likely include:
- title
- writer
- date
- source type
- tags
- short memo or excerpt preview

The structure can be:
- list
- card list
- reading table

But it should remain low-noise.

### 4. Optional curation shelf

If useful, the page can include a curated shelf related to the topic, such as:
- Start here
- Read together
- Closely related

This should not overpower the main topic record list.

### 5. Navigation cues

The page should help the user:
- go back to the main library
- jump into one specific record
- continue browsing nearby themes if appropriate

But avoid turning the page into a complex discovery dashboard.

## Design Goals

### A. Make one topic feel meaningful

Current risk:
- a tag page can easily feel like a raw filtered result

Design goal:
- make the topic feel intentional and worth exploring

### B. Keep reading-first hierarchy

The page should still feel like part of a reading product.
It should not look like metadata management.

### C. Make record scanning fast

A reader should be able to:
- scan multiple records quickly
- understand the tone of each record
- choose what to open next

### D. Preserve calmness

Even with more structure, the page should stay:
- warm
- reflective
- spacious

## Data Shown Per Record

Likely enough:
- title
- writer
- published date
- source type
- relevant tags
- short memo or excerpt preview

Do not overload every card with too many metadata fields.

## Design Direction

Keep and reinforce:
- warm paper-like palette
- serif-forward editorial typography
- quiet spacing rhythm
- subtle borders and low-noise surfaces
- literary, calm, confident tone

Avoid:
- generic filter/search result layouts
- dashboard-style sidebars
- giant taxonomy chips everywhere
- social-feed recommendation patterns
- heavy card chrome

Prefer:
- a thematic reading shelf
- elegant hierarchy
- soft emphasis
- calm discovery cues

## UX Priorities

- clear topic identity
- strong but subtle entry point
- quick record scanning
- natural onward reading
- desktop and mobile usability

## What To Decide In Design

Please propose design decisions for:
- how strong the topic title and intro should be
- whether the featured entry should be large or restrained
- how memo previews should appear in the topic list
- whether the topic page should include one curated shelf
- how the page should visually differ from the main library
- how much metadata is enough before the page becomes noisy

## Output Expectation

Produce desktop and mobile designs for:
- one dedicated topic hub page

The result should feel like:
- a natural extension of the `Personal Wiki`
- a richer form of tag exploration
- a page that helps one topic unfold into a reading path

## Tone

- thoughtful
- literary
- warm
- modern
- calm
- precise
