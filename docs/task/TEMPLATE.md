# Task Template

이 템플릿은 `docs/task/v0_X_[major-topic].md` 문서를 만들 때 사용한다.

원칙:
- 실제 구현 범위와 의사결정 기준을 고정한다.
- 아이디어 나열보다 실행 범위와 acceptance criteria를 분명하게 적는다.
- 테스트 절차 전체를 길게 복붙하지 말고 필요한 검증 포인트만 남긴다.

```md
# Personal Wiki v0.X Task Title

## Branch

- Working branch: `V0_X_[Major_Topic]`

## Goal

- 이번 작업의 목적을 한두 문장으로 적는다.

## Scope

- 포함할 작업을 적는다.

## Non-Goals

- 이번 작업에서 하지 않을 것을 적는다.

## Current Problems

- 현재 사용자 경험 또는 구현상 문제를 적는다.

## Proposed Changes

- 어떤 방식으로 해결할지 항목별로 적는다.

## Acceptance Criteria

- 완료 판단 기준을 구체적으로 적는다.

## Risks

- 작업 중 깨질 수 있는 점이나 주의할 점을 적는다.

## Verification

- 필요한 검증 명령이나 확인 포인트를 적는다.

## Related Docs

- 관련 idea, test guide, figma prompt, setup guide, PR 링크를 적는다.
```
